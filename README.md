# OpenLine Cognitive Immune Layer

A cognitive immune layer separates **intake** from **commit**.

Most systems protect themselves by narrowing intake. That makes them brittle.

This repo tests the opposite architecture:

```text
wide intake at the edge
strict verification at the core
portable receipts for every boundary crossing
```

Core line:

> A healthy system is porous at the edge and strict at the core.

## What this repo contains

This is a toy proof-of-architecture using verified Ulam-sequence boundary tests.

It does **not** prove consciousness, general AI safety, human psychology, institutional legitimacy, or universal governance theory.

It tests a narrower engineering claim:

> Preserved boundary records can help regulate future crossings.

The important distinction is **buffer vs. commit**.

A system can inspect a wide set of signals without letting every signal rewrite protected state.

## Architecture

```text
1. Permeable buffer
   Inspect more signals.

2. Verification quorum
   Require independent agreement before protected updates.

3. Commit barrier
   Only quorum-cleared updates alter the core.

4. Receipt memory
   Every crossing leaves a portable record.

5. Recursive controller
   Past boundary records help regulate future crossings.
```

## Product thesis

A weak system protects itself by refusing to look.

A stronger system can inspect more because it has a hardened write barrier.

```text
Open edges.
Strict commits.
Portable receipts.
```

## Claim boundary

This repo supports a toy-domain architecture claim:

> You can widen intake without corrupting the core if intake and commit are separated by strict quorum verification.

It does not claim zero false blocks in real systems.

Real systems have correlated witnesses, stale incentives, adversarial gaming, politics, measurement error, and institutional capture.

The practical target is to reduce false commits and false blocks, not promise perfection.

## Canon
